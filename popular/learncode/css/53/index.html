animation-play-state

animation-play-state xususiyati animatsiyani vaqtincha to'xtatish yoki davom ettirish imkonini beradi. Bu hover bo'lganda animatsiyani to'xtatish yoki JavaScript orqali boshqarish uchun juda foydali.

CSS
/* animation-play-state qiymatlari */
.running { animation-play-state: running; }  /* Animatsiya ishlaydi (default) */
.paused { animation-play-state: paused; }    /* Animatsiya to'xtatilgan */

/* Amaliy misol - hover bo'lganda to'xtatish */
.loader {
    animation: spin 1s linear infinite;
}
.loader:hover {
    animation-play-state: paused;  /* Sichqoncha olib borilganda to'xtaydi */
}

/* Murakkab misol - barcha animatsiyalarni to'xtatish */
.animation-paused * {
    animation-play-state: paused !important;
}
Senior biladi: animation-play-state: paused animatsiyani joriy holatida muzlatadi. Davom ettirilganda xuddi shu joydan boshlanadi.

will-change - Performans optimizatsiyasi

will-change xususiyati brauzerga qaysi CSS xususiyatlari o'zgarishini oldindan aytib beradi. Bu brauzerga kerakli optimizatsiyalarni oldindan qilish imkonini beradi va animatsiya silliqroq ishlaydi.

CSS
/* will-change - brauzerga oldindan xabar berish */
.animated-element {
    will-change: transform, opacity;  /* Bu xususiyatlar o'zgaradi */
    transition: transform 0.3s, opacity 0.3s;
}
.animated-element:hover {
    transform: scale(1.1);
    opacity: 0.8;
}

/* Noto'g'ri ishlatish - hamma narsaga will-change berish */
*, *::before, *::after {
    will-change: all;  /* ❌ BU NOTO'G'RI! Performansni pasaytiradi! */
}

/* To'g'ri ishlatish - faqat animatsiya bo'ladigan elementlarga */
.card {
    will-change: transform;  /* ✅ Faqat kerakli xususiyat */
    transition: transform 0.3s;
}
Muhim: will-change ni faqat haqiqatan ham animatsiya bo'ladigan elementlarga bering. Hammasiga berish xotirani ortiqcha sarflaydi va performansni pasaytiradi!

prefers-reduced-motion (Accessibility)

prefers-reduced-motion - bu media query bo'lib, foydalanuvchi operatsion tizimida "animatsiyalarni kamaytirish" sozlamasini yoqqan yoki yoqmaganligini tekshiradi. Professional dasturchi har doim accessibility ni hisobga olishi kerak!

CSS
/* Foydalanuvchi animatsiyalarni kamaytirishni so'ragan bo'lsa */
@media (prefers-reduced-motion: reduce) {
    *,
    *::before,
    *::after {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
        scroll-behavior: auto !important;
    }
}

/* Yoki aniq elementlar uchun */
@media (prefers-reduced-motion: reduce) {
    .loader {
        animation: none;  /* Animatsiyani butunlay o'chirish */
    }
    .animated-card {
        transition: none;  /* Transitionlarni o'chirish */
    }
}

/* no-preference - animatsiyalarni yoqish mumkin bo'lganda */
@media (prefers-reduced-motion: no-preference) {
    .hero-title {
        animation: slideIn 1s ease-out;
    }
}
Senior biladi: prefers-reduced-motion ni qo'llash nafaqat yaxshi amaliyot, balki ba'zi hollarda qonuniy talab hamdir (WCAG accessibility standartlari).

Murakkab @keyframes yozish

Professional animatsiyalar bir nechta xususiyatlarni bir vaqtda o'zgartirishni talab qiladi. Har bir foizda bir nechta xususiyatni o'zgartirish mumkin.

CSS
/* Murakkab animatsiya - bir nechta xususiyatlar */
@keyframes professionalEntrance {
    0% {
        opacity: 0;
        transform: translateY(-30px) scale(0.9);
        filter: blur(5px);
    }
    60% {
        opacity: 0.8;
        transform: translateY(10px) scale(1.02);
        filter: blur(2px);
    }
    100% {
        opacity: 1;
        transform: translateY(0) scale(1);
        filter: blur(0);
    }
}

/* Shimmer loading effekti */
@keyframes shimmer {
    0% { background-position: -200% 0; }
    100% { background-position: 200% 0; }
}
.shimmer {
    background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
    background-size: 200% 100%;
    animation: shimmer 1.5s infinite;
}

Animatsiya performansi - GPU vs CPU

Barcha CSS xususiyatlari bir xil tezlikda animatsiya qilinmaydi. Faqat transform va opacity GPU da ishlaydi va eng silliq animatsiyalarni beradi.

XususiyatQayerda ishlaydiPerformansTavsiya
transformGPU⭐⭐⭐⭐⭐Har doim ishlating
opacityGPU⭐⭐⭐⭐⭐Har doim ishlating
filterGPU (ba'zilari)⭐⭐⭐Ehtiyot bo'ling
background-colorCPU⭐⭐⭐Kerak bo'lganda
width/heightCPU⭐Iloji boricha ishlatmang
top/leftCPU⭐transform ishlating
box-shadowCPU⭐Ehtiyot bo'ling
CSS - Yaxshi vs Yomon
/* YOMON - CPU da ishlaydi, lag bo'ladi */
.bad-animation {
    transition: left 0.3s, top 0.3s, width 0.3s, height 0.3s;
}
.bad-animation:hover {
    left: 100px;
    top: 50px;
    width: 200px;
    height: 150px;
}

/* YAXSHI - GPU da ishlaydi, juda silliq */
.good-animation {
    transition: transform 0.3s, opacity 0.3s;
}
.good-animation:hover {
    transform: translate(100px, 50px) scale(1.2);
    opacity: 0.8;
}
