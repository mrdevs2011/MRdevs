Combinators (Kombinatorlar)

15-yanvar, 2021 CSS Asoslari O'rta daraja
Bu darsda nimalarni o'rganasiz?

CSS kombinatorlari elementlar orasidagi bog'liqlik asosida tanlash imkonini beradi. Bu darsda 4 ta asosiy kombinatorni: Descendant (bo'shliq), Child (>), Adjacent Sibling (+) va General Sibling (~) ni o'rganasiz.

Bu darsda nimalarni o'rganasiz?

  • Descendant combinator (bo'shliq) - avlodlar
  • Child combinator (>) - bevosita farzandlar
  • Adjacent sibling combinator (+) - keyingi qo'shni
  • General sibling combinator (~) - barcha qo'shnilar
  • Kombinatorlarni birgalikda ishlatish

Bu dars uchun maslahatlar

Belgilarni eslab qoling: Bo'shliq (avlod), > (farzand), + (birinchi qo'shni), ~ (barcha qo'shnilar). yodlash shart
Child (>) ko'proq ishlatiladi: Bevosita farzandlarni tanlash uchun > kombinatori juda foydali. muhim
Qo'shnilar DOM da bir xil darajada: + va ~ bir xil ota elementga ega bo'lgan elementlar uchun ishlaydi. amaliyot

Descendant Combinator (Bo'shliq)

Descendant combinator - ikkita selektor orasidagi bo'shliq. Ota element ichidagi barcha mos keladigan elementlarni (qanchalik chuqur bo'lishidan qat'i nazar) tanlaydi.

CSS 
/* div ichidagi barcha p elementlari */
div p {
    color: #2563eb;
}

/* .container ichidagi barcha a elementlari */
.container a {
    text-decoration: none;
}
HTML 
<div>
    <p>Birinchi daraja - tanlanadi</p>
    <section>
        <p>Ikkinchi daraja - tanlanadi</p>
        <div>
            <p>Uchinchi daraja - tanlanadi</p>
        </div>
    </section>
</div>
Qachon ishlatiladi? Ota element ichidagi barcha ma'lum turdagi elementlarni tanlash kerak bo'lganda, masalan, menyu ichidagi barcha havolalar.

Child Combinator (>)

Child combinator (>) - faqat bevosita birinchi darajali farzand elementlarni tanlaydi. Chuqurroqdagi nevaralar tanlanmaydi.

CSS 
/* div ning faqat bevosita farzand p lari */
div > p {
    font-weight: bold;
    color: #dc2626;
}

/* .menu ning bevosita farzand li lari */
.menu > li {
    display: inline-block;
}
HTML 
<div>
    <p>Birinchi daraja - tanlanadi ✅</p>
    <section>
        <p>Ikkinchi daraja - tanlanmaydi ❌</p>
    </section>
</div>
Descendant vs Child: div p barcha p larni tanlaydi, div > p faqat birinchi darajali p larni tanlaydi.

Adjacent Sibling Combinator (+)

Adjacent sibling combinator (+) - berilgan elementdan darhol keyin keladigan birinchi qo'shni elementni tanlaydi. Ikkala element bir xil ota elementga ega bo'lishi kerak.

CSS 
/* h1 dan keyin darhol keladigan p */
h1 + p {
    margin-top: 0;
    font-style: italic;
}

/* .error dan keyin darhol keladigan .message */
.error + .message {
    border-left-color: #dc2626;
}
HTML 
<h1>Sarlavha</h1>
<p>Birinchi paragraf - tanlanadi ✅</p>
<p>Ikkinchi paragraf - tanlanmaydi ❌</p>

General Sibling Combinator (~)

General sibling combinator (~) - berilgan elementdan keyin keladigan barcha qo'shni elementlarni tanlaydi. Ikkala element bir xil ota elementga ega bo'lishi kerak.

CSS 
/* h1 dan keyin keladigan barcha p lar */
h1 ~ p {
    color: #64748b;
}

/* .active dan keyin keladigan barcha .item lar */
.active ~ .item {
    opacity: 0.7;
}
HTML 
<h1>Sarlavha</h1>
<p>Birinchi paragraf - tanlanadi ✅</p>
<div>Ajratuvchi</div>
<p>Ikkinchi paragraf - tanlanadi ✅</p>

Kombinatorlar xulosasi

Kombinator Belgi Misol Nimani tanlaydi?
Descendant bo'shliq div p div ichidagi barcha p lar (har qanday chuqurlikda)
Child > div > p div ning faqat bevosita farzand p lari
Adjacent Sibling + h1 + p h1 dan darhol keyin keladigan birinchi p
General Sibling ~ h1 ~ p h1 dan keyin keladigan barcha p lar

Bilimingizni tekshiring

8-dars testi
1 div p qanday elementlarni tanlaydi?
2 ul > li qanday elementlarni tanlaydi?
3 h2 + p qanday elementni tanlaydi?
4 h2 ~ p qanday elementlarni tanlaydi?

Amaliy mashq

Vazifa: Kombinatorlardan foydalanib quyidagi vazifalarni bajaring.

  1. nav ichidagi barcha a elementlarini yashil rangda qiling (Descendant)
  2. nav ning faqat bevosita farzand a larini qalin qiling (Child)
  3. h1 dan keyin darhol keladigan p ni kursiv qiling (Adjacent Sibling)
  4. h2 dan keyin keladigan barcha p larni kulrang rangda qiling (General Sibling)

Namunaviy yechim:

/* 1. Descendant - barcha avlodlar */
nav a {
    color: #10b981;
}

/* 2. Child - faqat bevosita farzandlar */
nav > a {
    font-weight: bold;
}

/* 3. Adjacent Sibling - birinchi qo'shni */
h1 + p {
    font-style: italic;
}

/* 4. General Sibling - barcha qo'shnilar */
h2 ~ p {
    color: #64748b;
}
<nav>
    <a href="#">Bosh sahifa</a>
    <a href="#">Biz haqimizda</a>
    <div>
        <a href="#">Ichki havola</a>
    </div>
</nav>

<h1>Sarlavha</h1>
<p>Birinchi paragraf - kursiv bo'ladi</p>
<p>Ikkinchi paragraf - oddiy</p>

<h2>Ikkinchi sarlavha</h2>
<p>Paragraf 1 - kulrang</p>
<p>Paragraf 2 - kulrang</p>

Dars xulosasi

  • Descendant (bo'shliq): ota ichidagi barcha elementlar
  • Child (>): faqat bevosita farzandlar
  • Adjacent Sibling (+): darhol keyingi birinchi qo'shni
  • General Sibling (~): keyingi barcha qo'shnilar
  • Keyingi darsda: "Class"lar va qo'shma selektorlar
