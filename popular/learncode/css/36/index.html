rem nimaga nisbatan hisoblanadi?

rem (root em) birligi har doim root element (<html>) ning font-size iga nisbatan hisoblanadi. Bu em dan asosiy farqi!

CSS 
/* Root element - barcha rem lar shunga nisbatan */
html {
    font-size: 16px;        /* 1rem = 16px (brauzer default) */
}

/* rem har doim root ga nisbatan - ota element font-size i ta'sir qilmaydi! */
.parent {
    font-size: 20px;        /* Bu rem ga ta'sir qilmaydi! */
}

.child {
    font-size: 1.5rem;      /* 16px * 1.5 = 24px (root dan) */
    padding: 2rem;          /* 16px * 2 = 32px */
    margin: 1rem;           /* 16px * 1 = 16px */
    width: 20rem;           /* 16px * 20 = 320px */
}

/* em bilan solishtirish */
.child-em {
    font-size: 1.5em;       /* 20px * 1.5 = 30px (ota elementdan!) */
}

62.5% trick - 1rem = 10px

Brauzer default font-size 16px. 62.5% bersak, 16px * 0.625 = 10px bo'ladi. Bu hisoblashni juda osonlashtiradi!

CSS 
/* 62.5% trick - 1rem = 10px */
html {
    font-size: 62.5%;       /* 16px * 0.625 = 10px */
}

/* Endi hisoblash juda oson! */
body {
    font-size: 1.6rem;      /* 10px * 1.6 = 16px */
}

h1 {
    font-size: 4.8rem;      /* 10px * 4.8 = 48px */
    margin-bottom: 2rem;    /* 10px * 2 = 20px */
}

.container {
    max-width: 120rem;      /* 10px * 120 = 1200px */
    padding: 2rem;          /* 10px * 2 = 20px */
}

.card {
    padding: 2.5rem;        /* 25px */
    border-radius: 0.8rem;  /* 8px */
}
Senior biladi: 62.5% trick orqali rem bilan ishlash px bilan ishlashdek oson bo'ladi, lekin responsive afzalliklari saqlanib qoladi!

Responsive typography rem bilan

rem birligining eng katta afzalligi - media queries orqali butun sayt shriftini bir vaqtda o'zgartirish imkoniyati.

CSS 
/* Desktop (default) */
html {
    font-size: 62.5%;       /* 1rem = 10px */
}

/* Planshet */
@media (max-width: 900px) {
    html {
        font-size: 56.25%;  /* 1rem = 9px (16px * 0.5625 = 9px) */
    }
}

/* Mobil */
@media (max-width: 600px) {
    html {
        font-size: 50%;     /* 1rem = 8px (16px * 0.5 = 8px) */
    }
}

/* Barcha rem o'lchamlari avtomatik o'zgaradi! */
body {
    font-size: 1.6rem;      /* Desktop: 16px, Planshet: 14.4px, Mobil: 12.8px */
}

h1 {
    font-size: 4.8rem;      /* Desktop: 48px, Planshet: 43.2px, Mobil: 38.4px */
}

rem vs em - Qachon qaysi biri?

rem ishlatiladigan holatlar

  • Global font-size
  • Margin va padding
  • Width va height
  • Media queries
  • Kontekstdan qat'iy nazar bir xil o'lcham kerak bo'lganda

em ishlatiladigan holatlar

  • Komponent ichidagi spacing (padding, margin)
  • Element o'lchami uning font-size iga bog'liq bo'lganda
  • Media queries da (rem tavsiya etiladi)
  • Ikonkalar o'lchami matnga mos bo'lishi kerak bo'lganda
CSS 
/* Global spacing - rem */
.container {
    max-width: 120rem;
    padding: 2rem;
    margin: 4rem auto;
}

/* Komponent - em (font-size ga bog'liq) */
.btn {
    font-size: 1.6rem;
    padding: 0.8em 1.5em;   /* font-size ga proporsional */
    border-radius: 0.4em;
}

.btn-small {
    font-size: 1.4rem;      /* padding avtomatik kichiklashadi! */
}

.btn-large {
    font-size: 2rem;        /* padding avtomatik kattalashadi! */
}

Vazifa: rem birligidan foydalanib to'liq responsive sahifa yarating.

  1. html { font-size: 62.5%; } orqali 1rem = 10px qiling
  2. Barcha o'lchamlarni rem da bering (font-size, margin, padding, width)
  3. Media queries orqali mobil va planshet uchun html { font-size } ni o'zgartiring
  4. Karta komponenti ichidagi spacing uchun em ishlating
  5. Katta va kichik karta variantlarini yarating (font-size o'zgartirish orqali)
