Animations (1-qism)

28-fevral, 2021 Transitions va Animations O'rta daraja
Bu darsda nimalarni o'rganasiz?

CSS da @keyframes qoidasi va animation xususiyatlari yordamida murakkab, ko'p bosqichli animatsiyalar yaratishni o'rganasiz. Transition faqat ikki holat o'rtasida silliq o'tish yaratsa, animation orqali cheksiz ko'p holatli, avtomatik boshlanadigan, takrorlanuvchi animatsiyalar yaratish mumkin. Bu zamonaviy web dizaynning ajralmas qismidir.

Bu darsda nimalarni o'rganasiz?

  • @keyframes - animatsiya kadrlarini belgilash
  • animation-name - animatsiya nomini bog'lash
  • animation-duration - animatsiya davomiyligi
  • animation-timing-function - tezlik egri chizig'i
  • animation-delay - animatsiya boshlanish kechikishi
  • animation-iteration-count - takrorlanishlar soni
  • animation-direction - yo'nalish (normal, reverse, alternate)
  • animation-fill-mode - animatsiya tugagandan keyingi holat

@keyframes - Animatsiya kadrlari

@keyframes - bu CSS animatsiyasining eng asosiy qismi. U orqali siz animatsiyaning boshlanishi, o'rtasi va oxiridagi holatlarini (kadrlarini) belgilaysiz. Brauzer siz belgilagan kadrlar orasidagi barcha o'tishlarni avtomatik hisoblab, silliq animatsiya yaratadi.

Oddiy tushuntirish

@keyframes - bu xuddi kinoplyonka. Siz plyonkaga bir nechta rasm (kadr) qo'yasiz, brauzer esa ular orasidagi bo'shliqni to'ldirib, silliq video yaratadi.

CSS - @keyframes sintaksisi
/* @keyframes asosiy sintaksisi */
@keyframes animatsiyaNomi {
    0% {
        /* Animatsiya boshidagi stillar */
        opacity: 0;
        transform: translateX(-100px);
    }
    50% {
        /* Animatsiya o'rtasidagi stillar */
        opacity: 0.5;
        transform: translateX(50px);
    }
    100% {
        /* Animatsiya oxiridagi stillar */
        opacity: 1;
        transform: translateX(0);
    }
}

/* from va to kalit so'zlari bilan (0% va 100% o'rniga) */
@keyframes fadeIn {
    from { opacity: 0; }
    to { opacity: 1; }
}
Foizlar mantiqi: 0% - animatsiya boshi, 100% - animatsiya oxiri. 50% - yarmi. Istalgan foizni ishlatishingiz mumkin (25%, 75%). Brauzer siz belgilagan kadrlar orasidagi barcha qiymatlarni avtomatik interpolyatsiya qiladi.

animation-name va animation-duration

@keyframes yaratganingizdan so'ng, uni elementga qo'llash uchun animation-name (qaysi animatsiya) va animation-duration (qancha davom etishi) xususiyatlarini berishingiz shart. Bu ikki xususiyat animatsiya ishlashi uchun majburiy hisoblanadi.

CSS
/* Animatsiyani qo'llash - majburiy xususiyatlar */
.animated-element {
    animation-name: fadeIn;      /* @keyframes nomi */
    animation-duration: 2s;      /* 2 sekund davom etadi */
}

/* Agar duration berilmasa, animatsiya ISHLAMAYDI! */
.no-duration {
    animation-name: fadeIn;
    /* animation-duration berilmagan - animatsiya ko'rinmaydi! */
}
Muhim: animation-duration berilmasa yoki 0s bo'lsa, animatsiya ishlamaydi! Bu yangi o'rganuvchilar eng ko'p qiladigan xato.

animation-timing-function

animation-timing-function animatsiya tezligining vaqt bo'yicha qanday o'zgarishini belgilaydi. Xuddi transition-timing-function kabi ishlaydi, lekin butun animatsiya davomida yoki har bir kadrlar o'rtasida qo'llanilishi mumkin.

CSS
/* Asosiy timing funksiyalari */
.linear { animation-timing-function: linear; }        /* Bir xil tezlik */
.ease { animation-timing-function: ease; }            /* Default - sekin-tez-sekin */
.ease-in { animation-timing-function: ease-in; }      /* Sekin boshlanib tezlashadi */
.ease-out { animation-timing-function: ease-out; }    /* Tez boshlanib sekinlashadi */
.ease-in-out { animation-timing-function: ease-in-out; } /* Sekin-tez-sekin */

/* Custom cubic-bezier - o'zingizning egri chiziq */
.custom { animation-timing-function: cubic-bezier(0.68, -0.55, 0.27, 1.55); }

/* steps() - pog'onali animatsiya */
.steps { animation-timing-function: steps(5, end); }  /* 5 ta pog'onada */
Senior biladi: steps() funksiyasi sprite animatsiyalar yoki yuklanish indikatorlari uchun juda foydali. Har bir kadr aniq alohida ko'rinadi.

animation-delay

animation-delay animatsiya boshlanishidan oldin qancha vaqt kutish kerakligini belgilaydi. Manfiy qiymat berilsa, animatsiya darhol boshlanadi, lekin xuddi shu vaqt oldin boshlangandek davom etadi.

CSS
/* Musbat kechikish */
.delayed {
    animation-name: fadeIn;
    animation-duration: 2s;
    animation-delay: 1s;        /* 1 sekund kutib, keyin boshlanadi */
}

/* Manfiy kechikish - darhol boshlanadi, lekin oldinroq boshlangandek */
.negative-delay {
    animation-name: fadeIn;
    animation-duration: 2s;
    animation-delay: -0.5s;     /* Animatsiya 0.5s oldin boshlangandek, yarmidan davom etadi */
}

/* Staggered animatsiyalar - ketma-ket chiroyli effekt */
.stagger-item { animation: fadeIn 0.5s ease forwards; opacity: 0; }
.stagger-item:nth-child(1) { animation-delay: 0.1s; }
.stagger-item:nth-child(2) { animation-delay: 0.3s; }
.stagger-item:nth-child(3) { animation-delay: 0.5s; }
.stagger-item:nth-child(4) { animation-delay: 0.7s; }

animation-iteration-count

animation-iteration-count animatsiya necha marta takrorlanishini belgilaydi. Butun son yoki infinite (cheksiz) qiymatini olishi mumkin.

CSS
/* Aniq son */
.once { animation-iteration-count: 1; }           /* 1 marta (default) */
.twice { animation-iteration-count: 2; }          /* 2 marta */
.three-times { animation-iteration-count: 3; }    /* 3 marta */
.half { animation-iteration-count: 0.5; }         /* Yarim marta (50% da to'xtaydi) */

/* Cheksiz takrorlanish */
.infinite { animation-iteration-count: infinite; } /* To'xtovsiz takrorlanadi */
Ehtiyot bo'ling: infinite animatsiyalar foydalanuvchi diqqatini chalg'itishi mumkin. Faqat zarur hollarda (loader, progress bar) ishlating.

animation-direction

animation-direction animatsiya yo'nalishini belgilaydi. Bu ayniqsa infinite takrorlanuvchi animatsiyalarda juda foydali.

CSS
/* Yo'nalish qiymatlari */
.normal { animation-direction: normal; }        /* 0% → 100% (default) */
.reverse { animation-direction: reverse; }      /* 100% → 0% (teskari) */
.alternate { animation-direction: alternate; }  /* normal → reverse → normal → ... */
.alternate-reverse { animation-direction: alternate-reverse; } /* reverse → normal → ... */

/* alternate - yumshoq qaytish effekti */
@keyframes pulse {
    from { transform: scale(1); }
    to { transform: scale(1.1); }
}
.pulse {
    animation: pulse 1s ease infinite alternate;  /* Kattalashib-kichiklashib turadi */
}
alternate fokus: alternate yordamida bitta yo'nalishda animatsiya yozib, avtomatik qaytish effektini olish mumkin. Bu kodni ikki barobar qisqartiradi.

animation-fill-mode

animation-fill-mode animatsiya boshlanishidan oldin va tugagandan keyin element qanday ko'rinishda qolishini belgilaydi. Bu juda muhim xususiyat - busiz animatsiya tugagach, element avvalgi holatiga sakrab qaytadi!

CSS
/* fill-mode qiymatlari */
.none { animation-fill-mode: none; }          /* Default - hech qanday stillar qo'llanmaydi */
.forwards { animation-fill-mode: forwards; }  /* Animatsiya tugagach, oxirgi kadr stillari saqlanadi */
.backwards { animation-fill-mode: backwards; } /* Animatsiya boshlanishidan oldin birinchi kadr stillari qo'llanadi */
.both { animation-fill-mode: both; }          /* forwards + backwards - eng foydali! */

/* Amaliy misol - forwards ishlatilmasa */
@keyframes slideIn {
    from { opacity: 0; transform: translateX(-100px); }
    to { opacity: 1; transform: translateX(0); }
}

.box-without-fill {
    animation: slideIn 1s;
    /* Animatsiya tugagach, element sakrab avvalgi holatiga qaytadi! */
}

.box-with-fill {
    animation: slideIn 1s forwards;
    /* Animatsiya tugagach, oxirgi holatda (opacity: 1, translateX: 0) qoladi */
}
Eng ko'p xato: Aksariyat hollarda siz forwards yoki both ishlatishni xohlaysiz. Aks holda animatsiya tugagach, element sakrab avvalgi holatiga qaytadi!

Animation shorthand

animation shorthand barcha xususiyatlarni bir qatorda yozish imkonini beradi. Tartib muhim emas, lekin birinchi vaqt qiymati duration, ikkinchisi delay sifatida qabul qilinadi.

CSS
/* Shorthand sintaksisi */
/* animation: name duration timing-function delay iteration-count direction fill-mode; */

/* Asosiy shorthand */
.fade-in {
    animation: fadeIn 2s ease-out;
}

/* To'liq shorthand */
.complex {
    animation: slideIn 1.5s cubic-bezier(0.68, -0.55, 0.27, 1.55) 0.2s 3 alternate both;
}

/* Bir nechta animatsiya */
.multiple {
    animation: 
        fadeIn 1s ease-out,
        pulse 2s ease-in-out 1s infinite;
}

Transition vs Animation - qachon qaysi biri?

XususiyatTransitionAnimation
TetikHodisa kerak (:hover, :focus)Avtomatik boshlanadi
Yo'nalishFaqat ikki holat (A → B)Ko'p kadrli (A → B → C → ...)
TakrorlanishYo'qCheksiz yoki aniq son
MurakkablikOddiyMurakkab
Qachon ishlatiladiHover, oddiy o'zgarishlarLoader, e'tibor tortuvchi elementlar
Senior qoidasi: Oddiy hover effektlar uchun transition, murakkab avtomatik animatsiyalar uchun animation ishlating.

Amaliy mashq (Senior daraja)

Vazifa: Animations yordamida loader va e'tibor tortuvchi elementlar yarating.

  1. @keyframes yordamida spin (aylanish) animatsiyasi yarating
  2. Loader elementiga infinite va linear qo'llang
  3. pulse animatsiyasi yarating (scale o'zgarishi)
  4. alternate yo'nalishidan foydalaning
  5. Barcha animatsiyalarda forwards yoki both ishlating

Senior darajadagi yechim:

/* Loader - aylanish animatsiyasi */
@keyframes spin {
    from { transform: rotate(0deg); }
    to { transform: rotate(360deg); }
}
.loader {
    width: 50px; height: 50px;
    border: 5px solid #e2e8f0;
    border-top-color: #3b82f6;
    border-radius: 50%;
    animation: spin 1s linear infinite;
}

/* Pulse - e'tibor tortish */
@keyframes pulse {
    from { transform: scale(1); box-shadow: 0 0 0 0 rgba(59, 130, 246, 0.5); }
    to { transform: scale(1.05); box-shadow: 0 0 0 15px rgba(59, 130, 246, 0); }
}
.pulse-btn {
    animation: pulse 2s ease-out infinite;
}

/* Staggered fade-in */
@keyframes fadeInUp {
    from { opacity: 0; transform: translateY(30px); }
    to { opacity: 1; transform: translateY(0); }
}
.stagger-item {
    opacity: 0;
    animation: fadeInUp 0.5s ease-out forwards;
}
.stagger-item:nth-child(1) { animation-delay: 0.1s; }
.stagger-item:nth-child(2) { animation-delay: 0.3s; }
.stagger-item:nth-child(3) { animation-delay: 0.5s; }

Dars xulosasi (Senior bilishi kerak)

  • @keyframes - animatsiya kadrlarini belgilaydi (0%, 50%, 100% yoki from/to)
  • animation-name + animation-duration - animatsiya ishlashi uchun MAJBURIY
  • animation-fill-mode: forwards/both - animatsiya tugagach oxirgi holatda qolish uchun (eng ko'p unutiladi!)
  • animation-iteration-count: infinite - cheksiz takrorlash (loaderlar uchun)
  • animation-direction: alternate - qaytish effekti (pulse uchun)
  • Transition vs Animation: oddiy hover = transition, murakkab avtomatik = animation
  • Keyingi darsda: Animations (2-qism)
